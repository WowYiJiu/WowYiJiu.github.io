用Git手动同步上游分支代码

Git常用命令

Github同步上游分支代码

Github进行fork后如何与上游仓库同步：重新fork很省事，但不如反复练习版本合并。

1.查看远程仓库状态

2.添加upstream(即fork的原仓库)

3.查看是否添加成功

4.拉取上游仓库更新

5.切换到主分支

6.合并上游更新

7.将更新后的代码推送到自己的远程仓库

第四五六步可以合成一个命令$ git pull upstream master
